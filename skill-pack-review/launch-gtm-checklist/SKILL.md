# Launch / GTM Checklist

The portfolio's failure mode is "built but earns nothing." A launch isn't "we hit publish" — it's
"the right buyer can find it, the page converts, and we can measure whether it worked."

## T-minus (before go-live)
- [ ] **Discoverability done, not deferred** — keywords/categories/tags chosen via the right research skill (kdp / etsy / seo / youtube). Default metadata = a non-launch.
- [ ] **Conversion path ready** — listing/page passes landing-page-cro + web-design-polish; the primary action works end-to-end, tested live.
- [ ] **Price set on purpose** (pricing-strategy), net-of-fees computed.
- [ ] **Measurement wired** — the conversion/sale event is instrumented + a finance_event will record (analytics-finops); baseline noted.
- [ ] **Assets complete** — cover/thumbnail/tiles QA-passed (ship-it-qa-checklist), no placeholder.
- [ ] **Distribution queued** — the $0 owned channels for this asset drafted + ready (Author Central/Goodreads, Etsy→Pinterest, YouTube→Shorts/community).
- [ ] **Credentials cleared** — the board credential/approval gate is the #1 stall point; confirm green + not expired BEFORE calling launch ready.

## T-0 (board-executed)
- [ ] Board approves + publishes (agents never publish externally).
- [ ] Verify LIVE not just submitted — view the real public listing/URL: metadata, price, images, working CTA.
- [ ] Fire the queued owned-channel distribution.

## T-plus (first 2 weeks)
- [ ] Day 1–2: confirm the conversion event records real data; fix if silent.
- [ ] Capture the launch-window baseline (sales/CTR/impressions).
- [ ] Triage early signals: zero impressions = discoverability; clicks no buys = price/page; refunds = expectation mismatch → customer-feedback-loop.
- [ ] Report the outcome to analytics-finops with the single highest-leverage next action.

## Anti-patterns
Launching into the void (no distribution/measurement); "published" ≠ "launched"; deferring all SEO to "later"; launching blocked on an expired board confirmation nobody re-armed.
